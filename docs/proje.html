Türkiye’nin Elektriği: Saatin, Mevsimin ve Kaynağın Hikâyesi

  • Show All Code
  • Hide All Code
  • View Source

Proje Konusu

Elektrik, modern bir ekonominin görünmez omurgasıdır: ne zaman, ne kadar ve hangi kaynaktan üretildiği; sanayinin çarklarından evlerin akşam ışıklarına kadar her şeyi belirler. Bu projede Türkiye’nin elektrik sistemini saatlik çözünürlükte ele alıyor ve üç temel soruyu yanıtlıyoruz:

  1. Saat ve mevsim: Elektriği günün hangi saatinde, yılın hangi döneminde daha çok tüketiyoruz? Talep nasıl bir ritim izliyor?
  2. Kaynak: Bu elektrik hangi kaynaklardan üretiliyor? Yenilenebilir kaynakların (rüzgâr, güneş, hidro) payı zaman içinde ve gün içinde nasıl değişiyor?
  3. Fiyat: Piyasa Takas Fiyatı (PTF) talep ve üretim kompozisyonuyla nasıl ilişkili? Özellikle güneşin bol olduğu öğle saatlerinde fiyat baskılanıyor mu? (Bu, enerji ekonomisinde “merit-order etkisi” olarak bilinir.)

Veriler EPİAŞ Şeffaflık Platformu’ndan alınmıştır; herkese açık ve ticari olmayan kullanıma uygundur.

Veri ve Ön İşleme

Analizde EPİAŞ Şeffaflık Platformu’ndan indirilen, saatlik çözünürlükte üç veri seti kullanılmıştır:

  • data/tuketim.xlsx — Gerçek zamanlı tüketim (MWh)
  • data/uretim.xlsx — Gerçek zamanlı üretim, kaynak bazlı (doğalgaz, barajlı, rüzgâr, güneş, linyit vb.; her biri ayrı sütun, MWh)
  • data/ptf.xlsx — Gün Öncesi Piyasası Piyasa Takas Fiyatı / PTF (TL/MWh)

Veriler platformun web arayüzünden Excel olarak indirilmiş, depo içindeki data/ klasörüne konulmuştur. Aşağıdaki adımlarda dosyalar R’a aktarılır, tarih/saat bilgisi ayrıştırılır ve üç set tek bir saatlik ana tabloda (master) birleştirilir.

Code 
# Tarih sütununu esnek biçimde POSIXct'e çeviren yardımcı fonksiyon.
# (EPİAŞ exportları kimi zaman düz metin, kimi zaman Excel tarihi getirir.)
parse_tarih <- function(x) {
  if (inherits(x, "POSIXct")) return(x)
  if (inherits(x, "Date"))    return(as_datetime(x))
  lubridate::parse_date_time(
    as.character(x),
    orders = c("Ymd HMS", "Ymd HM", "dmY HMS", "dmY HM",
               "Ymd THMS", "Ymd", "dmY"),
    tz = "Europe/Istanbul"
  )
}

# Bir tablodaki ilk sayısal sütunu (varsa anahtar kelimeye öncelik vererek) seçer.
ilk_sayisal <- function(df, anahtar = NULL) {
  num <- df |> dplyr::select(where(is.numeric))
  if (!is.null(anahtar)) {
    esles <- names(num)[str_detect(names(num), regex(anahtar, ignore_case = TRUE))]
    if (length(esles) > 0) return(num[[esles[1]]])
  }
  num[[1]]
}

# Bir tabloda anahtar kelimeye uyan ilk sütunu döndürür (yoksa NA).
pick_col <- function(df, key) {
  hit <- names(df)[str_detect(names(df), regex(key, ignore_case = TRUE)) &
                     names(df) != "tarih"]
  if (length(hit)) df[[hit[1]]] else rep(NA_real_, nrow(df))
}
Code 
# --- 1) Tüketim ---
tuketim_raw <- read_excel("data/tuketim.xlsx") |> clean_names()
names(tuketim_raw)[1] <- ".tarih_raw"           # ilk sütun = tarih
tuketim <- tibble(
  tarih       = parse_tarih(tuketim_raw$.tarih_raw),
  tuketim_mwh = ilk_sayisal(dplyr::select(tuketim_raw, -.tarih_raw), "tuket")
)

# --- 2) PTF (fiyat) ---
ptf_raw <- read_excel("data/ptf.xlsx") |> clean_names()
names(ptf_raw)[1] <- ".tarih_raw"
ptf <- tibble(
  tarih  = parse_tarih(ptf_raw$.tarih_raw),
  ptf_tl = ilk_sayisal(dplyr::select(ptf_raw, -.tarih_raw), "tl")  # TL/MWh sütunu
)

# --- 3) Üretim (kaynak bazlı -> uzun format) ---
uretim_raw <- read_excel("data/uretim.xlsx") |> clean_names()
names(uretim_raw)[1] <- ".tarih_raw"
uretim_long <- uretim_raw |>
  mutate(tarih = parse_tarih(.tarih_raw)) |>
  dplyr::select(-.tarih_raw) |>
  pivot_longer(cols = where(is.numeric),
               names_to = "kaynak", values_to = "uretim_mwh") |>
  # "toplam/genel" gibi özet sütunlarını analiz dışında bırak
  filter(!str_detect(kaynak, regex("toplam|total|genel", ignore_case = TRUE)))

# Yenilenebilir kaynakları işaretle
yenilenebilir_anahtar <- "ruzgar|wind|gunes|solar|jeotermal|biyokutle|baraj|akarsu|hidro|atik"
uretim_long <- uretim_long |>
  mutate(yenilenebilir = str_detect(kaynak, regex(yenilenebilir_anahtar,
                                                  ignore_case = TRUE)))

# Her saat için toplam üretim ve yenilenebilir payı
uretim_ozet <- uretim_long |>
  group_by(tarih) |>
  summarise(
    toplam_uretim        = sum(uretim_mwh, na.rm = TRUE),
    yenilenebilir_uretim = sum(uretim_mwh[yenilenebilir], na.rm = TRUE),
    .groups = "drop"
  ) |>
  mutate(yenilenebilir_pay = 100 * yenilenebilir_uretim / toplam_uretim)

# Kaynakları geniş formatta da tut (korelasyon için)
kaynak_genis <- uretim_long |>
  group_by(tarih, kaynak) |>
  summarise(uretim_mwh = sum(uretim_mwh, na.rm = TRUE), .groups = "drop") |>
  pivot_wider(names_from = kaynak, values_from = uretim_mwh)
Code 
ay_isim <- c("Oca","Şub","Mar","Nis","May","Haz",
             "Tem","Ağu","Eyl","Eki","Kas","Ara")

master <- tuketim |>
  left_join(ptf,         by = "tarih") |>
  left_join(uretim_ozet, by = "tarih") |>
  filter(!is.na(tarih)) |>
  mutate(
    tarih_gun  = as_date(tarih),
    saat       = hour(tarih),
    ay_no      = month(tarih),
    ay         = factor(ay_isim[ay_no], levels = ay_isim),
    hafta_gunu = wday(tarih, week_start = 1),
    hafta_sonu = if_else(hafta_gunu >= 6, "Hafta Sonu", "Hafta İçi"),
    mevsim     = case_when(
      ay_no %in% c(12, 1, 2) ~ "Kış",
      ay_no %in% c(3, 4, 5)  ~ "İlkbahar",
      ay_no %in% c(6, 7, 8)  ~ "Yaz",
      TRUE                   ~ "Sonbahar"
    )
  )

Analiz ve Bulgular

Gün İçi Tüketim Ritmi

Elektrik talebi gün içinde sabit değildir; insan yaşamının ritmini birebir takip eder. Saatlik ortalamalarla bu profili, hafta içi ve hafta sonunu ayırarak inceleyelim.

Code 
master |>
  group_by(saat, hafta_sonu) |>
  summarise(ort = mean(tuketim_mwh, na.rm = TRUE), .groups = "drop") |>
  ggplot(aes(saat, ort, color = hafta_sonu)) +
  geom_line(linewidth = 1) +
  geom_point(size = 1.6) +
  scale_x_continuous(breaks = seq(0, 23, 3)) +
  scale_y_continuous(labels = label_comma()) +
  scale_color_manual(values = c("Hafta İçi" = "#2c7fb8", "Hafta Sonu" = "#d95f0e")) +
  labs(title = "Gün İçi Tüketim Profili",
       x = "Saat", y = "Ort. Tüketim (MWh)", color = NULL)

Profil tipik olarak çift tepeli bir yapı gösterir: sabah iş/okul başlangıcıyla bir yükseliş, gece yarısına doğru en düşük seviye. Hafta içi talebin, sanayi ve ticaretin aktif olması nedeniyle hafta sonundan belirgin biçimde yüksek seyretmesi beklenir; bu fark özellikle gündüz saatlerinde açılır.

Saat × Ay Isı Haritası

Saat ve ay boyutlarını tek bir görselde birleştiren ısı haritası, talebin “ne zaman” yoğunlaştığını çarpıcı biçimde özetler.

Code 
master |>
  group_by(ay, saat) |>
  summarise(ort = mean(tuketim_mwh, na.rm = TRUE), .groups = "drop") |>
  ggplot(aes(ay, saat, fill = ort)) +
  geom_tile() +
  scale_fill_viridis_c(option = "C", labels = label_comma()) +
  scale_y_continuous(breaks = seq(0, 23, 3)) +
  labs(title = "Saat × Ay Ortalama Tüketim Isı Haritası",
       x = NULL, y = "Saat", fill = "MWh")

Isı haritasındaki en parlak (yüksek tüketim) bölgeler, yaz aylarının gündüz/öğleden sonra saatlerinde ve kış aylarının akşam saatlerinde yoğunlaşma eğilimindedir. Bu desen, yazın soğutma, kışın ısıtma ve aydınlatma kaynaklı talebin saat içindeki farklı zamanlamalarını ortaya koyar.

Mevsimsellik

Aylara göre günlük tüketim dağılımını kutu grafiğiyle incelersek, hem ortalama düzeyi hem de ay içindeki değişkenliği birlikte görebiliriz.

Code 
master |>

  summarise(gunluk = mean(tuketim_mwh, na.rm = TRUE), .groups = "drop") |>
  ggplot(aes(ay, gunluk)) +
  geom_boxplot(fill = "#a6bddb", outlier.alpha = 0.3) +
  scale_y_continuous(labels = label_comma()) +
  labs(title = "Aylara Göre Günlük Tüketim Dağılımı",

İncelenen Ocak–Nisan 2026 döneminde kış aylarının (Ocak–Şubat) ısıtma ve aydınlatma kaynaklı yüksek talebi dikkat çekerken, ilkbahara (Mart–Nisan) doğru ılıman havayla birlikte tüketimin kademeli olarak gerilediği görülür. Geniş kutular, ay içindeki gün-gün değişkenliğe işaret eder.

Güneşin Gün İçi İmzası

Güneş üretimi, doğası gereği yalnızca gündüz saatlerinde gerçekleşir ve öğle civarında zirve yapar. Bu “kubbe” şeklindeki profil, güneşin sisteme katkısının zamanlamasını gösterir.

Code 
uretim_long |>
  filter(str_detect(kaynak, regex("gunes|solar", ignore_case = TRUE))) |>
  mutate(saat = hour(tarih)) |>
  group_by(saat) |>
  summarise(ort = mean(uretim_mwh, na.rm = TRUE), .groups = "drop") |>
  ggplot(aes(saat, ort)) +
  geom_area(fill = "#fec44f", alpha = 0.6) +
  geom_line(color = "#d95f0e", linewidth = 1.1) +
  scale_x_continuous(breaks = seq(0, 23, 3)) +
  scale_y_continuous(labels = label_comma()) +
  labs(title = "Güneş Üretiminin Gün İçi Profili",
       x = "Saat", y = "Ort. Üretim (MWh)")

Üretim sabah doğuşla yükselir, öğle saatlerinde (≈12-14) zirve yapar ve akşam batışıyla sıfıra iner. Bu zamanlamanın, az sonra göreceğimiz fiyat davranışı üzerinde doğrudan etkisi vardır.

Fiyat (PTF) Davranışı

kor_src <- tibble(
  tarih    = kaynak_genis$tarih,
  Dogalgaz = pick_col(kaynak_genis, "dogalgaz|gas"),
  Baraj    = pick_col(kaynak_genis, "baraj"),
  Ruzgar   = pick_col(kaynak_genis, "ruzgar|wind"),
  Gunes    = pick_col(kaynak_genis, "gunes|solar")
)

kor_veri <- master |>
  dplyr::select(tarih, tuketim_mwh, ptf_tl, toplam_uretim, yenilenebilir_pay) |>
  left_join(kor_src, by = "tarih") |>
  dplyr::select(-tarih) |>
  rename(Tuketim = tuketim_mwh, PTF = ptf_tl,
         `Toplam Üretim` = toplam_uretim, `Yenilenebilir %` = yenilenebilir_pay)

cm <- cor(kor_veri, use = "pairwise.complete.obs")
corrplot(cm, method = "circle", type = "upper",
         addCoef.col = "black", number.cex = 0.7,
         tl.col = "black", tl.srt = 45)

Korelasyon matrisinde tüketim ile toplam üretim arasında güçlü pozitif bir ilişki beklenir (talep arzı çeker). PTF’nin tüketimle pozitif, yenilenebilir payıyla negatif yönde ilişkili çıkması ise yukarıda anlattığımız iki mekanizmayı (talep baskısı ve merit-order etkisi) doğrular niteliktedir.

Sonuçlar ve Çıkarımlar

Bu çalışma, Türkiye’nin elektrik sistemini saatlik veriyle üç eksende inceledi ve şu temel bulgulara ulaştı:

  • Talebin bir ritmi var. Tüketim gün içinde çift tepeli (sabah ve akşam) bir profil izliyor; hafta içi hafta sonundan yüksek. İncelenen kış–ilkbahar döneminde kış ayları daha yüksek talep gösteriyor.
  • Üretimde termik kaynaklar baskın, ama yenilenebilir büyüyor. Doğalgaz ve kömür ağırlığını korurken, barajlı hidro mevsimsel; rüzgâr ve güneşin payı artış eğiliminde.
  • Güneşin zamanlaması fiyatı etkiliyor. Güneş öğle saatlerinde zirve yapıyor; yenilenebilir payının yüksek olduğu saatlerde PTF baskılanıyor (merit-order etkisi). Talep arttığında ise fiyat yükseliyor.

Bu tablo, talep yönetimi ve depolama yatırımlarının değerini ortaya koyuyor: gündüz bol ve ucuz olan yenilenebilir enerjiyi akşam pik saatlerine taşıyabilmek, hem maliyetleri hem de termik kaynaklara bağımlılığı azaltabilir. Saatlik fiyat sinyallerine duyarlı bir tüketim (örneğin esnek sanayi yükleri, akıllı şarj) sistemin verimliliğini artırma potansiyeline sahiptir.

# Proje Konusu

Elektrik, modern bir ekonominin görünmez omurgasıdır: ne zaman, ne kadar ve hangi kaynaktan üretildiği; sanayinin çarklarından evlerin akşam ışıklarına kadar her şeyi belirler. Bu projede **Türkiye'nin elektrik sistemini saatlik çözünürlükte** ele alıyor ve üç temel soruyu yanıtlıyoruz:

1.  **Saat ve mevsim:** Elektriği günün hangi saatinde, yılın hangi döneminde daha çok tüketiyoruz? Talep nasıl bir ritim izliyor?
2.  **Kaynak:** Bu elektrik hangi kaynaklardan üretiliyor? Yenilenebilir kaynakların (rüzgâr, güneş, hidro) payı zaman içinde ve gün içinde nasıl değişiyor?
3.  **Fiyat:** Piyasa Takas Fiyatı (PTF) talep ve üretim kompozisyonuyla nasıl ilişkili? Özellikle güneşin bol olduğu öğle saatlerinde fiyat baskılanıyor mu? (Bu, enerji ekonomisinde **"merit-order etkisi"** olarak bilinir.)

Veriler [EPİAŞ Şeffaflık Platformu](https://seffaflik.epias.com.tr/)'ndan alınmıştır; herkese açık ve ticari olmayan kullanıma uygundur.

# Veri ve Ön İşleme

Analizde EPİAŞ Şeffaflık Platformu'ndan indirilen, saatlik çözünürlükte üç veri seti kullanılmıştır:

-   **`data/tuketim.xlsx`** — Gerçek zamanlı tüketim (MWh)
-   **`data/uretim.xlsx`** — Gerçek zamanlı üretim, kaynak bazlı (doğalgaz, barajlı, rüzgâr, güneş, linyit vb.; her biri ayrı sütun, MWh)
-   **`data/ptf.xlsx`** — Gün Öncesi Piyasası Piyasa Takas Fiyatı / PTF (TL/MWh)

Veriler platformun web arayüzünden Excel olarak indirilmiş, depo içindeki `data/` klasörüne konulmuştur. Aşağıdaki adımlarda dosyalar R'a aktarılır, tarih/saat bilgisi ayrıştırılır ve üç set tek bir **saatlik ana tablo**da (`master`) birleştirilir.

```{r}
#| label: yardimci-fonksiyonlar

# Tarih sütununu esnek biçimde POSIXct'e çeviren yardımcı fonksiyon.
# (EPİAŞ exportları kimi zaman düz metin, kimi zaman Excel tarihi getirir.)
parse_tarih <- function(x) {
  if (inherits(x, "POSIXct")) return(x)
  if (inherits(x, "Date"))    return(as_datetime(x))
  lubridate::parse_date_time(
    as.character(x),
    orders = c("Ymd HMS", "Ymd HM", "dmY HMS", "dmY HM",
               "Ymd THMS", "Ymd", "dmY"),
    tz = "Europe/Istanbul"
  )
}

# Bir tablodaki ilk sayısal sütunu (varsa anahtar kelimeye öncelik vererek) seçer.
ilk_sayisal <- function(df, anahtar = NULL) {
  num <- df |> dplyr::select(where(is.numeric))
  if (!is.null(anahtar)) {
    esles <- names(num)[str_detect(names(num), regex(anahtar, ignore_case = TRUE))]
    if (length(esles) > 0) return(num[[esles[1]]])
  }
  num[[1]]
}

# Bir tabloda anahtar kelimeye uyan ilk sütunu döndürür (yoksa NA).
pick_col <- function(df, key) {
  hit <- names(df)[str_detect(names(df), regex(key, ignore_case = TRUE)) &
                     names(df) != "tarih"]
  if (length(hit)) df[[hit[1]]] else rep(NA_real_, nrow(df))
}
```

```{r}
#| label: veri-okuma

# --- 1) Tüketim ---
tuketim_raw <- read_excel("data/tuketim.xlsx") |> clean_names()
names(tuketim_raw)[1] <- ".tarih_raw"           # ilk sütun = tarih
tuketim <- tibble(
  tarih       = parse_tarih(tuketim_raw$.tarih_raw),
  tuketim_mwh = ilk_sayisal(dplyr::select(tuketim_raw, -.tarih_raw), "tuket")
)

# --- 2) PTF (fiyat) ---
ptf_raw <- read_excel("data/ptf.xlsx") |> clean_names()
names(ptf_raw)[1] <- ".tarih_raw"
ptf <- tibble(
  tarih  = parse_tarih(ptf_raw$.tarih_raw),
  ptf_tl = ilk_sayisal(dplyr::select(ptf_raw, -.tarih_raw), "tl")  # TL/MWh sütunu
)

# --- 3) Üretim (kaynak bazlı -> uzun format) ---
uretim_raw <- read_excel("data/uretim.xlsx") |> clean_names()
names(uretim_raw)[1] <- ".tarih_raw"
uretim_long <- uretim_raw |>
  mutate(tarih = parse_tarih(.tarih_raw)) |>
  dplyr::select(-.tarih_raw) |>
  pivot_longer(cols = where(is.numeric),
               names_to = "kaynak", values_to = "uretim_mwh") |>
  # "toplam/genel" gibi özet sütunlarını analiz dışında bırak
  filter(!str_detect(kaynak, regex("toplam|total|genel", ignore_case = TRUE)))

# Yenilenebilir kaynakları işaretle
yenilenebilir_anahtar <- "ruzgar|wind|gunes|solar|jeotermal|biyokutle|baraj|akarsu|hidro|atik"
uretim_long <- uretim_long |>
  mutate(yenilenebilir = str_detect(kaynak, regex(yenilenebilir_anahtar,
                                                  ignore_case = TRUE)))

# Her saat için toplam üretim ve yenilenebilir payı
uretim_ozet <- uretim_long |>
  group_by(tarih) |>
  summarise(
    toplam_uretim        = sum(uretim_mwh, na.rm = TRUE),
    yenilenebilir_uretim = sum(uretim_mwh[yenilenebilir], na.rm = TRUE),
    .groups = "drop"
  ) |>
  mutate(yenilenebilir_pay = 100 * yenilenebilir_uretim / toplam_uretim)

# Kaynakları geniş formatta da tut (korelasyon için)
kaynak_genis <- uretim_long |>
  group_by(tarih, kaynak) |>
  summarise(uretim_mwh = sum(uretim_mwh, na.rm = TRUE), .groups = "drop") |>
  pivot_wider(names_from = kaynak, values_from = uretim_mwh)
```

```{r}
#| label: ana-tablo

ay_isim <- c("Oca","Şub","Mar","Nis","May","Haz",
             "Tem","Ağu","Eyl","Eki","Kas","Ara")

master <- tuketim |>
  left_join(ptf,         by = "tarih") |>
  left_join(uretim_ozet, by = "tarih") |>
  filter(!is.na(tarih)) |>
  mutate(
    tarih_gun  = as_date(tarih),
    saat       = hour(tarih),
    ay_no      = month(tarih),
    ay         = factor(ay_isim[ay_no], levels = ay_isim),
    hafta_gunu = wday(tarih, week_start = 1),
    hafta_sonu = if_else(hafta_gunu >= 6, "Hafta Sonu", "Hafta İçi"),
    mevsim     = case_when(
      ay_no %in% c(12, 1, 2) ~ "Kış",
      ay_no %in% c(3, 4, 5)  ~ "İlkbahar",
      ay_no %in% c(6, 7, 8)  ~ "Yaz",
      TRUE                   ~ "Sonbahar"
    )
  )
```

```{r}
  geom_smooth(method = "lm", color = "firebrick", se = TRUE) +
  scale_x_continuous(labels = label_comma()) +
  scale_y_continuous(labels = label_comma()) +
  labs(title = "Tüketim ile PTF İlişkisi",
       x = "Saatlik Tüketim (MWh)", y = "PTF (TL/MWh)")
```

Ardından **fiyat–yenilenebilir payı**: rüzgâr ve güneş gibi yakıt maliyeti sıfıra yakın kaynakların payı arttığında, pahalı termik santrallere ihtiyaç azalır ve fiyatın düşmesi beklenir. Bu, enerji ekonomisindeki **merit-order (liyakat sırası) etkisidir.**

```{r}
#| label: ptf-yenilenebilir
master |>
  ggplot(aes(yenilenebilir_pay, ptf_tl)) +
  geom_point(alpha = 0.12, color = "seagreen") +
  geom_smooth(method = "lm", color = "firebrick", se = TRUE) +
  scale_y_continuous(labels = label_comma()) +
  labs(title = "Yenilenebilir Payı ile PTF İlişkisi (Merit-Order Etkisi)",
       x = "Yenilenebilir Pay (%)", y = "PTF (TL/MWh)")
```

Aşağıdaki basit doğrusal model, bu ilişkilerin yönünü ve büyüklüğünü sayısal olarak özetler.

```{r}
#| label: regresyon
model <- lm(ptf_tl ~ tuketim_mwh + yenilenebilir_pay, data = master)
summary(model)
```

## Korelasyon Matrisi

Son olarak, temel değişkenler arasındaki doğrusal ilişkileri tek bir görselde toplayalım.

```{r}
#| label: korelasyon
kor_src <- tibble(
  tarih    = kaynak_genis$tarih,
  Dogalgaz = pick_col(kaynak_genis, "dogalgaz|gas"),
  Baraj    = pick_col(kaynak_genis, "baraj"),
  Ruzgar   = pick_col(kaynak_genis, "ruzgar|wind"),
  Gunes    = pick_col(kaynak_genis, "gunes|solar")
)

kor_veri <- master |>
  dplyr::select(tarih, tuketim_mwh, ptf_tl, toplam_uretim, yenilenebilir_pay) |>
  left_join(kor_src, by = "tarih") |>
  dplyr::select(-tarih) |>
  rename(Tuketim = tuketim_mwh, PTF = ptf_tl,
         `Toplam Üretim` = toplam_uretim, `Yenilenebilir %` = yenilenebilir_pay)

cm <- cor(kor_veri, use = "pairwise.complete.obs")
corrplot(cm, method = "circle", type = "upper",
         addCoef.col = "black", number.cex = 0.7,
         tl.col = "black", tl.srt = 45)
```

Korelasyon matrisinde tüketim ile toplam üretim arasında güçlü pozitif bir ilişki beklenir (talep arzı çeker). PTF'nin tüketimle pozitif, yenilenebilir payıyla negatif yönde ilişkili çıkması ise yukarıda anlattığımız iki mekanizmayı (talep baskısı ve merit-order etkisi) doğrular niteliktedir.

# Sonuçlar ve Çıkarımlar

Bu çalışma, Türkiye'nin elektrik sistemini saatlik veriyle üç eksende inceledi ve şu temel bulgulara ulaştı:

-   **Talebin bir ritmi var.** Tüketim gün içinde çift tepeli (sabah ve akşam) bir profil izliyor; hafta içi hafta sonundan yüksek. İncelenen kış–ilkbahar döneminde kış ayları daha yüksek talep gösteriyor.
-   **Üretimde termik kaynaklar baskın, ama yenilenebilir büyüyor.** Doğalgaz ve kömür ağırlığını korurken, barajlı hidro mevsimsel; rüzgâr ve güneşin payı artış eğiliminde.
-   **Güneşin zamanlaması fiyatı etkiliyor.** Güneş öğle saatlerinde zirve yapıyor; yenilenebilir payının yüksek olduğu saatlerde PTF baskılanıyor (merit-order etkisi). Talep arttığında ise fiyat yükseliyor.

Bu tablo, **talep yönetimi ve depolama** yatırımlarının değerini ortaya koyuyor: gündüz bol ve ucuz olan yenilenebilir enerjiyi akşam pik saatlerine taşıyabilmek, hem maliyetleri hem de termik kaynaklara bağımlılığı azaltabilir. Saatlik fiyat sinyallerine duyarlı bir tüketim (örneğin esnek sanayi yükleri, akıllı şarj) sistemin verimliliğini artırma potansiyeline sahiptir.
